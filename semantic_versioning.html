Semantic Versioning (SemVer) provides a formal specification dictating how version numbers are assigned and incremented. By giving a name and clear definition to versioning, it becomes easy to communicate your intentions to the users of your software.

MAJOR.MINOR.PATCH

The Increment Rules

This section breaks down the foundational rules of SemVer 2.0.0. It defines exactly when you must increment the Major, Minor, or Patch numbers, establishing the contract between software creators and consumers regarding API stability.

X

MAJOR

Incompatible API changes.

  • • Increment when breaking backwards compatibility.
  • • Resets Minor and Patch to 0.
  • • Version 0.y.z is for initial unstable development.
Y

MINOR

Backwards compatible features.

  • • Increment for new functionality that does not break API.
  • • Increment if API functionality is deprecated.
  • • Resets Patch to 0.
Z

PATCH

Backwards compatible bug fixes.

  • • Increment for internal changes fixing incorrect behavior.
  • • Must NOT affect the public API.

Numbered Capture Groups

Official Regex (ECMAScript/PCRE)

^(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)(?:-((?:0|[1-9]\d*|\d*[a-zA-Z-][0-9a-zA-Z-]*)(?:\.(?:0|[1-9]\d*|\d*[a-zA-Z-][0-9a-zA-Z-]*))*))?(?:\+([0-9a-zA-Z-]+(?:\.[0-9a-zA-Z-]+)*))?$

This regex ensures no leading zeroes on numeric identifiers and strictly validates allowed ASCII characters.

Based on the Semantic Versioning 2.0.0 Specification authored by Tom Preston-Werner.
